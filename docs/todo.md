2. フロントエンド側の409ハンドリング
実装場所: PecusApiClient.ts

Axiosインターセプターで 409 ステータスをキャッチして、ユーザーにフレンドリーなエラーメッセージを表示。

ユーザー操作
   ↓
クライアント検証 (Zod)
   ↓
Server Action 呼び出し
   ↓
サーバー検証 + API呼び出し
   ↓
[成功] → redirect() で画面遷移
   ↓
[409 Conflict] ← DbUpdateConcurrencyException
   ↓
ユーザーにダイアログ表示
"別のユーザーが変更しました"
   ↓
[再試行] [キャンセル]
   ↓
[再試行] → 最新データを自動リロード → フォーム再入力
[キャンセル] → 編集内容を破棄 → 一覧画面に戻る
