この画面のエンドユーザー向けマニュアルをMarkdownにまとめます（docs/help/配下に作成）。
- Zod スキーマがある場合は、必須/任意など入力要件はスキーマ優先で記載すること（過剰な説明は不要）。
- 見出しに項目番号を付けないこと（❌ "### 1."、⭕️ "### タイトル"）。
- 保存時の同時実行・競合説明は不要。トースト通知など一時的なUI挙動も記載しない。
- 不明点は ○○○○ として残してよい（例: 用途：○○○○）。
- 推奨セクション例（必要に応じて取捨選択可）:
	- タイトル行（H1）
	- この画面でできること
	- 画面構成 / 主な機能
	- フィルター・入力項目（必須/任意・文字数など）
	- 操作の流れ（簡潔に、番号なし）
	- 注意事項 / 制約
- スタイル:
    - 箇条書きを中心に簡潔に。固有名はUI表記に合わせる。
    - 入力項目やボタン名は太字にする。
