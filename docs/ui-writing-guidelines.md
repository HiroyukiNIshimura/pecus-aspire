# UI ライティングガイドライン

## 概要

このドキュメントは、Pecus Aspire のフロントエンド UI における文言の統一ルールを定義します。
ユーザーに一貫した体験を提供し、操作の安心感を高めることを目的としています。

---

## 基本原則

### 1. ユーザーを主語にする
- ❌ `システムがデータを削除します`
- ✅ `このデータを削除します`

### 2. 影響を具体的に説明する
- ❌ `削除しますか？`
- ✅ `削除すると、このユーザーはワークスペースにアクセスできなくなります。`

### 3. 丁寧語を使うが、堅すぎない
- ❌ `削除いたしますがよろしいでしょうか？`
- ✅ `削除しますか？`

### 4. 専門用語を避ける
- ❌ `トランザクションがロールバックされました`
- ✅ `処理中にエラーが発生しました。もう一度お試しください。`

## モーダル・ダイアログ

### タイトル

| 操作 | パターン | 例 |
|------|---------|---|
| 削除確認 | `〇〇を削除しますか？` | `メンバーを削除しますか？` |
| 変更確認 | `〇〇を変更しますか？` | `ロールを変更しますか？` |
| 追加確認 | `〇〇を追加しますか？` | `メンバーを追加しますか？` |
| 危険な操作 | `〇〇の削除`（タイトルを error 色に） | `ワークスペースの削除` |

### 本文

#### 通常の確認

```
{対象名} を {操作} します。
```

例：
```
山田太郎 のロールを変更します。
```

#### 危険な操作（削除・無効化など）

```
⚠️ この操作を行うと、{影響の説明}。
```

例：
```
⚠️ この操作を行うと、このユーザーはワークスペースにアクセスできなくなります。
```

#### 取り消し不可の操作

```
⚠️ この操作は取り消せません。
```

### 警告レベル

| レベル | 使用場面 | 視覚表現 |
|-------|---------|---------|
| **情報** | 単純な確認 | 通常テキスト |
| **注意** | 権限変更など | ⚠️ + `text-warning` |
| **危険** | 削除・破壊的操作 | ⚠️ + `text-error` + `alert-error` |

### 確認入力

重要な操作には確認入力を求める：

| 操作 | 確認入力 |
|------|---------|
| ワークスペース削除 | ワークスペースコードの入力 |
| メンバー削除 | メールアドレスの入力 |
| アカウント削除 | 「削除する」と入力 |

ラベル例：
```
削除を確認するため、メールアドレスを入力してください
```

## ボタン

### ラベルは動詞形にする

| ❌ 避ける | ✅ 推奨 |
|----------|--------|
| `削除` | `削除する` |
| `変更` | `変更する` |
| `追加` | `追加する` |
| `OK` | `確認する` / 具体的な動詞 |

### キャンセルボタン

- 常に `キャンセル` を使用
- 位置は左側（確定ボタンの前）

### 処理中の表示

```tsx
{isLoading ? (
  <>
    <span className="loading loading-spinner loading-sm"></span>
    削除中...
  </>
) : (
  '削除する'
)}
```

パターン：
- `削除中...`
- `変更中...`
- `追加中...`
- `保存中...`
- `送信中...`

## 通知メッセージ（notify）

### 成功メッセージ

```
{対象} を {操作} しました。
```

例：
- `ワークスペースを更新しました。`
- `山田太郎 をワークスペースに追加しました。`
- `山田太郎 のロールを変更しました。`

### エラーメッセージ

#### API エラー（具体的な原因がわかる場合）

```
{操作} に失敗しました。
```

例：
- `メンバーの追加に失敗しました。`
- `ワークスペースの更新に失敗しました。`

#### 通信エラー（原因不明の場合）

```
サーバーとの通信でエラーが発生しました。
```

※ `persistent: true` で永続表示にする

#### バリデーションエラー

```
{具体的な問題}
```

例：
- `ファイルサイズは5MB以下にしてください`
- `対応している画像形式: JPEG, PNG, GIF, WebP`

### 警告メッセージ

```
{注意事項}
```

例：
- `このユーザーは既にメンバーです。`

## フォーム

### ラベル

```
{項目名}
```

例：`ワークスペース名`、`メールアドレス`

### プレースホルダー

```
{入力例}
```

例：`例：プロジェクトA`、`example@email.com`

### バリデーションエラー

```
{項目名}は{制約}で入力してください。
```

例：
- `ワークスペース名は100文字以内で入力してください。`
- `メールアドレスの形式が正しくありません。`

### 必須項目

ラベルの後に赤いアスタリスクを付ける：

```tsx
<label className="label">
  <span className="label-text font-semibold">ワークスペース名</span>
  <span className="label-text-alt text-error">*</span>
</label>
```

## 状態表示

### 空の状態

```
{対象}がありません
```

例：
- `メンバーが登録されていません`
- `ワークスペースがありません`

### ローディング

```
読み込み中...
```

または、スピナーアイコンのみ（テキストなし）

### エラー状態

```
{対象}の読み込みに失敗しました
```

例：`ワークスペース情報の読み込みに失敗しました`

## ロール・権限の表示

| ロール | 日本語表示 | バッジスタイル |
|-------|-----------|--------------|
| Owner | オーナー | `badge-outline badge-warning` |
| Member | メンバー | `badge-outline badge-success` |
| Viewer | 閲覧者 | `badge-outline badge-light` |

### 権限変更時の説明文

```tsx
// オーナー権限を付与する場合
⚠️ オーナー権限を付与すると、このユーザーはワークスペースの設定を変更できるようになります。

// 閲覧者に変更する場合
閲覧者は、ワークスペースのコンテンツを閲覧のみ可能です。
```

## アイコンの使用

| 用途 | アイコン |
|------|---------|
| 警告 | ⚠️ または `icon-[solar--alert-triangle]` |
| エラー | `icon-[solar--x]` |
| 成功 | `icon-[solar--check]` |
| 情報 | `icon-[solar--info-circle]` |
| 削除 | `icon-[solar--trash]` |
| 編集 | `icon-[solar--pencil]` |
| 追加 | `icon-[solar--plus]` |

## チェックリスト

新しい UI を作成する際は、以下を確認してください：

- [ ] タイトルは `〇〇を{動詞}しますか？` 形式か
- [ ] 危険な操作には ⚠️ と影響の説明があるか
- [ ] ボタンラベルは動詞形か（`削除する`）
- [ ] 処理中の状態が表示されるか
- [ ] 成功/エラー通知が統一されているか
- [ ] 確認入力が必要な操作に実装されているか
