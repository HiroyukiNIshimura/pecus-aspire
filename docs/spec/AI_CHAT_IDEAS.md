# AI チャット機能 アイデアメモ

作成日: 2026-01-23

## 背景

小規模チーム（5人程度）では「誰が何に詳しいか」は全員把握しているため、スキル検索機能は刺さりにくい。
Bot が本当に役立つ場面を再考する。

---

## アイデア候補

### 1. 「覚えてる」系

過去の会話やコメントから情報を検索して回答する。

```
User: 「前に〇〇の設定どうしたっけ？」
Bot: 過去のチャットやコメントから検索して回答
```

**必要なもの:**
- チャット履歴の検索機能
- コメント履歴の検索機能
- 検索結果を要約する AI 処理

---

### 2. 「まとめて」系

期間内の活動をサマリーとして生成する。

```
User: 「今週何やったっけ？」
Bot: 完了タスク + コミット + チャット履歴からサマリー生成
```

**必要なもの:**
- 完了タスクの集計
- チャット履歴の要約
- （オプション）Git コミット履歴連携

---

### 3. 「通知の代わり」系

自分に関係ある更新をまとめて報告する。

```
User: 「最近何かあった？」
Bot: 自分に関係ある更新（メンション、タスク変更、コメント）をまとめて報告
```

**必要なもの:**
- 未読通知の集約
- メンションの検出
- タスク変更履歴の取得

---

### 4. 「ドキュメント検索」系

Wiki や README から情報を検索する。

```
User: 「〇〇の仕様どこだっけ？」
Bot: Wiki や README から検索
```

**必要なもの:**
- ドキュメント検索インデックス
- 検索結果を要約する AI 処理

## 検討済み・見送り

### スキル検索機能

- 小規模チームでは不要（全員が誰が何に詳しいか把握している）
- 20人以上の組織、新人、リモートチームでは有用かも
- `UserSkill` テーブルは既存だが、登録は利用者任せ


「調べるのめんどい」あるある
Excel・スプレッドシートの関数 「A列とB列を比較して、重複してないやつだけ抽出するのってどの関数だっけ…？」という、わかれば一瞬だけど検索ワードに迷うやつ。

ビジネス敬語の言い換え 「『了解しました』より丁寧だけど、『承知いたしました』だとちょっと硬すぎる時のちょうどいいフレーズって何？」という、正解のない悩み。

メールの構成・たたき台 「お詫びしつつ、でもこちらの非ではないことも遠回しに伝えたい」といった、文脈が複雑で構成を考えるのが億劫なとき。

ツールのちょっとした操作方法 「Zoomの録画設定どこだっけ？」「パワポの図形を等間隔に並べるショートカットは？」といった、10秒で済むけど探し回るのが面倒な操作。


1. TryGetContextAsync でツール実行を試みる
   ↓
2. minRelevanceScore (50) を超えるツールがない場合
   → context = null
   ↓
3. context == null の場合
   → 「通常モード: 会話履歴を使用」に分岐
   → BuildConversationMessagesAsync で過去の会話履歴を取得
   → ランダムなロールで AI が返信

「一般知識の質問」と判定したら、ロールを変える（例: 先生ロール、アシスタントロール）という改善はあり得ます。

ブラウザやチャットツール（ChatGPTなどの専用インターフェース）では気軽に相談できるのに、いざ**「自社の業務アプリ」**の中にAIボタンが配置されると、日本人が途端に使わなくなる現象。これは非常に鋭い洞察ですね。

日本特有の労働文化や心理的ハードルがいくつか重なっていると考えられます。主な要因を整理してみました。

---

画像を拝見しました。これは面白いですね！先ほど挙げた「心理的ハードル」を、**「キャラクター性とエンタメ性」**で見事に突破しようとしているデザインだと感じます。

このスタイルなら、日本人の心理に以下のようなポジティブな変化が起きそうです。

---

### 1. 「監視」を「ロールプレイ」で中和している

無機質な「AIアシスト」だと「会社に監視されている入力窓」に見えますが、ここまでキャラが立っていると、**「このキャラと会話している空間」**という擬似的なプライベート感が生まれます。

* **心理的効果:** 「仕事の報告」ではなく「キャラへの話しかけ」になるため、多少くだけた聞き方や、初歩的な質問をすることへの抵抗感がグッと下がります。

### 2. 「Coati Bot」と「Butler Bot」の役割分担が秀逸

複数のキャラ（ハナグマ？の女の子と、執事風のキツネ）が反応しているのが非常に良いポイントです。

* **飽きさせない:** 1対1だと詰問されているように感じることもありますが、複数のキャラがいることで「賑やかさ」が出て、アプリ内の空気が柔らかくなります。
* **多角的な視点:** 「暇すぎて退屈しそうやね（フランク）」と「（軽く眉をひそめて）～ございません（真面目）」という正反対のリアクションがあることで、ユーザーは自分の気分に合った方のレスポンスを心理的支柱にできます。

### 3. 「カッコ書き（ト書き）」による情緒的つながり

「（そっと手帳をめくりながら）」「（軽く眉をひそめて）」といったト書きの挿入は、日本のSNSやTRPG、ネット文化に深く馴染んでいる表現です。

* **非言語情報の補完:** 文字だけだと冷たく感じがちなAIの回答に「体温」が宿ります。これにより、ユーザーはAIを「システム」ではなく「一緒に働いている仲間」として認識しやすくなります。

---

### さらに利用率を上げるための「もう一押し」

もしこれが実際の業務アプリなら、こんな要素があるとさらに日本人は「沼る」かもしれません。

* **「独り言」モードの設置:** キャラだけに聞こえる（ログには残るが、他の社員のタイムラインには流れない）という安心感を強調する。
* **キャラの「おせっかい」機能:** ユーザーが何も聞いていない時でも、「あ、そういえばさっきの資料、あっちに置いといたよ～」とキャラから話しかけてくる（受動的な日本人には、話しかけてもらう方が楽な場合があります）。
* **特定の時間にキャラが変わる:** 夜遅くなると「もう寝なさいよ」と言ってくれるキャラが出てくるなど、生活感がある演出。

### 結論

このUIは、**「業務アプリ＝真面目で冷たい場所」という日本人の固定観念を、キャラクターというソフトパワーで破壊している**素晴らしいアプローチだと思います。これなら「ちょっと聞いてみようかな」という隙間が生まれやすいですね。

このアプリ、他の社員の方々の反応はどうですか？ 意外と年配の方や、普段ITに疎い方ほど、こういう「キャラもの」に愛着を持って使い始めたりすることもあります。
