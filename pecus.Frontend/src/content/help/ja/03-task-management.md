# タスク管理機能

ワークスペース内のタスクを効率的に管理するための機能について説明します。

![画面](/help/images/1000006.webp)

## タスクのステータス
タスクは以下のステータスを持ちます。一覧画面でフィルタリングして表示できます。

- **未完了 (Active)**: 現在進行中または着手待ちのタスクです。
- **完了 (Completed)**: 作業が終了したタスクです。
- **破棄 (Discarded)**: 不要になった、または中止されたタスクです。

## タスク詳細とコメント
タスク詳細画面では、進捗状況の更新やメンバー間のコミュニケーションが行えます。

![画面](/help/images/1000007.webp)

### コメント機能
目的に応じてコメントの種類を使い分けることで、円滑なコミュニケーションを支援します。

- **通常**: 一般的な連絡や報告に使用します。
- **メモ**: 個人の備忘録として使用します。
- **助けて**: 作業に行き詰まった際、他のメンバーに支援を求めるために使用します。
- **返事が欲しい**: 特定のメンバーからの回答が必要な場合に使用します。
- **リマインダー**: 特定の日時に通知を受け取りたい場合に使用します。
- **督促**: 期限が迫っているタスクに対して注意を促す場合に使用します。

**コメントの種類は担当者とその他のメンバーとでは上記の中で利用可能な項目が変化します**

## タスクフローマップ
タスク間の依存関係（先行・後続）を視覚的に確認できる機能です。
アイテム詳細画面の「フローマップ」ボタンから表示できます。

![画面](/help/images/1000008.webp)

### 主な機能
**クリティカルパス**:
最も時間がかかる作業経路です。このパス上のタスクが遅れると、全体の完了が遅れます。
- 赤枠で強調表示されます。
- 「212日」のように残り合計期間が表示されます。
- 矢印（→）で先行タスクから後続タスクへの流れを示します。

**依存チェーン**:
クリティカルパス以外の依存関係のあるタスク群です。

- 期間の長い順に並んでいます。
- 各チェーンには「○ステップ」「○日」の情報が表示されます。

**独立タスク（依存関係なし）**:
他のタスクに依存せず、いつでも着手できるタスクです。

- 並行して作業を進められます。
- 先行タスクを設定すると、依存チェーンに組み込まれます。

### タスクカードの見方

| 表示 | 意味 |
|---|---|
| 🟢 緑のプログレスバー | 進捗率（0〜100%） |
| 🔴 緊急 バッジ | 優先度が「緊急」のタスク |
| ⚠️ 黄色の日付 | 先行タスクの期限が後続より遅い（スケジュール矛盾） |
| 👤 アイコン | 担当者 |
| 📅 日付 | 期限日 |

### 上部の統計情報

| 項目 | 説明 |
|---|---|
| 総タスク | アイテム内のタスク総数 |
| 着手可能 | 今すぐ作業を始められるタスク数 |
| 待機中 | 先行タスクの完了を待っているタスク数 |
| 完了 | 完了済みのタスク数 |
| 合計期間 | 未完了タスクの推定所要日数（土日除く） |
| 担当者 | 関わっている担当者の人数 |

💡 ヒント
- 権限があればタスクカードをクリックすると、タスクの詳細を編集できます。
- 「先行タスク」を設定すると、タスク間の依存関係を作れます。
- クリティカルパス上のタスクを優先的に進めると、全体の遅延を防げます。
- 左サイドメニューでアイテムを切り替えるとフローマップをアイテムを横断して確認できます。

### 活用方法
- **着手可能タスクの特定**: 先行タスクが完了し、すぐに着手できるタスクを一目で把握できます。
- **ボトルネックの発見**: 多くのタスクが依存しているタスク（ボトルネック）を特定し、優先的に対応する計画を立てられます。

## 担当者の負荷確認
タスク作成・編集時に担当者を割り当てる際、その担当者の期限日周辺のタスク負荷状況を確認できます（合計期間➗担当者）。
無理のないスケジュール調整に役立ちます。

## タスク期間の計算方法
タスクの所要期間は、以下のルールで自動計算されます。

### 基本ルール
**推定工数が設定されている場合**

入力した推定工数を「1日 = 8時間」として日数に換算します。
例：16時間 → 2日、4時間 → 0.5日

**推定工数が未設定の場合**

開始日から期限日までの営業日（土日を除く）で計算します。
祝日は営業日としてカウントされます。

### 開始日の決め方
開始日は以下の優先順位で自動決定されます：

- 開始日が設定されている場合 → その日付を使用
- 先行タスクがある場合 → 先行タスクの期限日を使用
- どちらもない場合 → タスク作成日を使用

💡 ポイント
完了済み・破棄済みのタスクは期間計算から除外されます。
より正確な期間を表示するには、見積もり工数を入力することをおすすめします。
先行タスクの期限日が後続タスクの期限日より後の場合、スケジュールの矛盾として警告が表示されます。
